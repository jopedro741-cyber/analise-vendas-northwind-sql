# 📊 Projeto de Análise de Vendas — Northwind (SQL)

## 📌 Sobre o Projeto

Este projeto tem como objetivo realizar uma análise de vendas utilizando o banco de dados **Northwind**, aplicando consultas SQL para responder perguntas de negócio relacionadas a **clientes, produtos, faturamento, categorias e comportamento de vendas**.

O foco do projeto foi desenvolver **raciocínio analítico**, modelagem de consultas e extração de insights utilizando **PostgreSQL**.

---

## 🎯 Objetivos da Análise

Responder perguntas de negócio como:

- Quais produtos são mais vendidos?
- Quais produtos geram maior faturamento?
- Quais clientes geram mais receita?
- Quais países possuem maior volume de pedidos?
- Qual o ticket médio dos pedidos?
- Quais produtos nunca foram vendidos?
- Como as vendas evoluíram ao longo dos anos?
- Quais categorias geram maior faturamento?

## 📈 Queries Desenvolvidas

### 1. Produtos mais vendidos
Identificação dos produtos com maior quantidade vendida.

### 2. Produtos que mais faturam
Análise dos produtos com maior geração de receita.

### 3. Clientes com maior volume de pedidos
Identificação dos clientes com maior número de compras.

### 4. Países com maior volume de pedidos
Análise dos países com mais pedidos realizados.

### 5. Ticket médio dos pedidos
Cálculo do ticket médio das vendas.

### 6. Clientes com maior receita
Identificação dos clientes com maior faturamento.

### 7. Produtos sem vendas
Análise de produtos sem movimentação comercial.

### 8. Categorias que mais faturam
Identificação das categorias mais lucrativas.

### 9. Evolução de vendas por ano
Análise temporal do faturamento.

### 10. Produtos premium
Identificação de produtos com maior valor agregado.

### 11. Clientes sem compras
Análise de clientes sem pedidos registrados.

### 12. Top países por faturamento
Identificação dos países com maior geração de receita.

### 13. Ticket médio por cliente
Análise do valor médio gasto por cliente.

### 14. Dashboard-ready query
Criação de uma tabela analítica pronta para uso no Power BI.
